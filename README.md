# 🚗 AI-Driven Car Insurance Telegram Bot

An intelligent, state-based Telegram bot designed to guide users through a seamless, 3-step comprehensive car insurance purchasing process. Built with **.NET 10**, C#, and powered entirely by **OpenAI's advanced LLM and Vision models**.

This project fulfills and significantly exceeds the requirements of the "Telegram Bot for Car Insurance Sales" test task by introducing a robust Finite State Machine (FSM), strict security guardrails, and dynamic multilingual capabilities.

| **1. Bot Setup** | ✔️ Handled via `/start`. The AI introduces itself, explains the 3-step process, and asks for the first document. |

## 🏗️ Architectural Decisions & Advanced Features

### 1. Mindee API vs. OpenAI Vision (Addressing Requirement #3)
The task allowed for simulating Mindee with a mock API or integrating it directly. While a `MockDocumentRecognitionService` is provided in the codebase, the primary implementation uses `OpenAiDocumentRecognitionService` via the **OpenAI Vision API**. 
**Why this approach is superior:**
* **Unified Ecosystem:** Eliminates the need for multiple third-party API keys and dependencies. Both conversational logic and OCR are handled by OpenAI.
* **The "Cat Detector" (Image Validation):** Unlike standard OCRs that simply fail on invalid images, OpenAI Vision actively classifies the image. If a user uploads a selfie, a cat, or a landscape, the bot gracefully detects it, makes a polite/humorous comment (e.g., *"What a cute Chihuahua, but I can't insure it"*), and asks for the actual document.
* **Multilingual OCR:** Perfectly extracts Cyrillic, Latin, and special characters regardless of the document's origin country.

### 2. Finite State Machine (FSM) & Prompt Composition Pattern
The bot relies on a strict FSM (`UserState` enum) tracked via an `InMemoryUserSessionStorage`. 
Instead of sending a massive, monolithic prompt to OpenAI, the system dynamically concatenates a `Prompt_Base.txt` (core persona, formatting rules) with highly specific `Prompt_[State].txt` files. This minimizes token usage and forces the LLM to focus only on the logic required for the current step.

### 3. C# Guardrails (Security)
To prevent the LLM from hallucinating or skipping steps (e.g., a user trying to bypass the document upload by just typing "Give me the policy"), the `MessageHandler` implements hardcoded C# Guardrails. If the AI suggests advancing the state without the required `[PHOTO_RECEIVED]` tags, the C# layer intercepts, logs a `[SECURITY]` warning, and forces the bot to stay in the waiting state.

### 4. Multilingual Matching & Personalization
The AI detects the user's language dynamically. The C# backend parses the `detectedLanguage` from the AI's JSON output, saves it to the `UserSession`, and injects it into future prompts. The bot also uses the user's Telegram First Name, gracefully switching to their Passport Name once the document is extracted.

### 5. The "Pause" Rule
If users ask questions about the insurance terms, pricing, or the algorithm at any point, the AI pauses the workflow, answers the question thoroughly using its injected `INSURANCE PROCESS KNOWLEDGE`, and politely asks the user to provide the currently expected document.

## 🚀 Deployment & Hosting

The application is containerized using **Docker** and hosted on **Render** (Free Tier).

* **Infrastructure:** The bot runs in a Linux container environment.
* **Webhooks:** The system uses Telegram Webhooks for instant message delivery instead of polling. 
* **Cold Start Note:** Since the project uses Render's Free Tier, the instance may "spin down" after 15 minutes of inactivity. If the bot doesn't respond immediately, please allow **up to 50 seconds** for the "cold start" to complete. Subsequent interactions will be near-instant.

## 🔄 Detailed Bot Workflow

1. **`New`**: User sends `/start`. Bot outlines the 3-step process.
2. **`WaitingForPassport`**: Bot waits for an image. Non-image inputs or invalid images (cats/food) are rejected. Valid images are processed via Vision API.
3. **`ConfirmingPassport`**: Bot presents extracted data. Accepts only Yes/No. "No" reverts to `WaitingForPassport`.
4. **`WaitingForVehicleDoc`**: Bot waits for the Vehicle Registration Document.
5. **`ConfirmingVehicleDoc`**: Bot presents extracted vehicle data. Accepts only Yes/No.
6. **`AwaitingPriceAgreement`**: Bot offers the fixed 100 USD price. Accepts only Yes/No.
7. **`Finished`**: Bot generates a highly formatted, locally translated Insurance Policy. The session is closed. Any further input prompts the user to type `/start` to begin anew.

## 💻 Setup Instructions & Dependencies

### Prerequisites
* **.NET 10 SDK**
* A Telegram Bot Token from [@BotFather](https://t.me/BotFather)
* An OpenAI API Key with access to Vision models (e.g., `gpt-4o-mini`, `gpt-5.4`)

### Secure Configuration (User Secrets)
This project follows strict security best practices. API keys are managed via the .NET Secret Manager, ensuring they are not hardcoded or accidentally pushed to source control.

1. Clone the repository and navigate to the API project:
    ```bash
    git clone <repository_url>
    cd CarInsuranceBot.API
    ```
2. Initialize User Secrets:
    ```bash
    dotnet user-secrets init
    ```
3. Add your API keys securely:
    ```bash
    dotnet user-secrets set "TelegramSettings:BotToken" "YOUR_TELEGRAM_BOT_TOKEN"
    dotnet user-secrets set "OpenAI:ApiKey" "YOUR_OPENAI_API_KEY"
    ```
4. *(Optional)* The OpenAI model is configured in `appsettings.json`. You can easily swap between models (default is `gpt-4o-mini` for speed and cost-efficiency) without recompiling the code.
    ```json
    {
      "OpenAI": {
        "Model": "gpt-4o-mini" 
        }
    }
    ```
5. Running the Application:
    ```bash
    dotnet build
    dotnet run
    ```

### Manual Webhook Setup (For Cloud Deployment)
If deploying to a new environment, remember to set the Telegram Webhook manually via browser:
#### [https://api.telegram.org/bot<YOUR_TOKEN>/setWebhook?url=<YOUR_RENDER_URL>/api/bot](https://api.telegram.org/bot<YOUR_TOKEN>/setWebhook?url=<YOUR_RENDER_URL>/api/bot)
